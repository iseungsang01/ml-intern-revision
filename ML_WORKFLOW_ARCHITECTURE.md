# KSTAR CES Prediction 자율 R&D 파이프라인 및 모델 워크플로우 명세서

본 문서는 KSTAR CES(Charge Exchange Spectroscopy) 데이터 예측을 위해 구축된 **다중 에이전트 기반 자율 R&D 루프(Autonomous Multi-Agent R&D Loop)**와 **Multimodal 딥러닝 예측 모델**의 현재 워크플로우를 상세히 설명하고, 각 파이프라인 구성의 물리적/공학적 근거(Rationale)를 정리합니다.

---

## 1. 딥러닝 예측 모델 워크플로우 (Multimodal CES Predictor)

기존 CES 장비가 가진 낮은 시간 해상도 한계를 극복하기 위해, BES, ECEI, Mirnov Coil(MC) 등의 고해상도 다중 진단 데이터를 융합하여 CES 주요 파라미터($T_i$, $V_{rot}$)를 예측합니다.

### 1.1 입력 데이터 파이프라인
*   **결측치 엄격 필터링(Strict NaN Handling)**: 모든 진단 장치의 데이터가 온전히 존재하는 시점의 샘플만 사용합니다.
    *   **근거**: 선형 보간 등으로 '가짜 데이터'를 생성하면 플라즈마의 순간적이고 미세한 동역학(예: Type III ELM, 섭동)을 왜곡할 위험이 크기 때문입니다.
*   **특성 추출(Feature Extraction)**: LSTM 등의 RNN 계열 대신 **1D CNN(Convolutional Neural Network)**을 사용합니다.
    *   **근거**: 엄격한 결측치 제거로 인해 시간 간격(Time-steps)이 불규칙해집니다. 연속적 시계열성에 의존하는 LSTM은 불규칙한 시간 간격에서 성능이 저하되므로, 지역적(Local) 시간 패턴 추출에 강점이 있는 1D CNN을 각 진단 장치 모듈에 일괄 적용합니다.

### 1.2 모델 아키텍처 (Late Fusion)
1.  **독립적 특성 추출**: BES(9채널), ECEI(4채널), MC(2채널), 시간 정보(dt) 각각에 대해 개별적인 1D CNN 블록을 통과시켜 64차원(시간은 16차원)의 특징 벡터를 추출합니다.
2.  **후기 융합(Late Fusion)**: 추출된 각 특징 벡터를 마지막 단에서 연결(Concatenate, 총 208차원)하여 Fully Connected Layer를 거쳐 $T_i$와 $V_{rot}$를 최종 예측합니다.
    *   **근거**: 다중 진단 데이터를 입력단에서 섣불리 평균 내거나 합치면 각 장치가 관측하는 고유의 공간적/물리적 특성을 잃어버릴 위험이 있습니다. 독립된 CNN 모듈을 통해 고유의 특성을 먼저 추상화한 뒤 융합하는 Late Fusion이 물리적 특징 보존에 유리합니다.

## 2. 자율 ML 연구 워크플로우 (Autonomous Multi-Agent R&D Loop)

본 프로젝트는 단순한 코드 어시스턴트를 넘어, 최적의 KSTAR 가상 센서 도출을 위해 스스로 실험하고 개선하는 **AI 과학자(AI Scientist)** 구조를 채택했습니다. 

단일 에이전트가 코딩과 평가를 모두 수행하는 대신, **3개의 전문화된 에이전트**가 협력하여 R&D 루프를 돌게 됩니다.

### 2.1 구성 요소 및 역할
1.  **Evaluation Agent (평가 에이전트)**
    *   **역할**: 주어진 모델 및 파이프라인(`train.py`)을 실행하여 학습을 진행하고, 검증 손실(Validation Loss) 및 각종 성능 지표를 수집합니다.
2.  **Briefing Agent (브리핑 에이전트)**
    *   **역할**: 여러 번의 실험 런(Run)에 걸친 성능 기록(History)을 추적하고 요약합니다. 현재 모델이 정체기(Plateau)에 빠졌는지, 아니면 여전히 수렴 중인지 판단합니다.
3.  **Researcher Agent (연구 에이전트)**
    *   **역할**: 브리핑 에이전트의 리포트를 분석하여 다음 실험 방향을 결정하고 실제 코드(`model.py` 등)를 수정합니다.

### 2.2 파이프라인 근거 (Rationale)
*   **관심사의 분리(Separation of Concerns)**:
    *   코드 구현, 실험 지표 추적, 새로운 아키텍처 가설 수립은 서로 완전히 다른 인지적 맥락(Cognitive context)을 요구합니다. 역할을 분리함으로써 LLM이 겪는 컨텍스트 혼란을 방지하고 각자의 역할에 집중하여 성능을 극대화할 수 있습니다.
*   **정체기 극복(Plateau Recovery)**:
    *   일반적인 자동화 스크립트는 단순히 하이퍼파라미터만 조정하며 Local Minima(지역 최솟값)에 빠지기 쉽습니다. Briefing Agent가 여러 번의 실험 흐름을 거시적으로 조망함으로써, 미세 조정으로 한계에 부딪혔을 때 Researcher Agent에게 'CNN 기반 모델을 폐기하고 Transformer 계열로 완전히 변경하라'는 식의 파격적이고 과감한 아키텍처 전환을 강제할 수 있습니다.
*   **완전 자율성(True Autonomy)**:
    *   실험이 끝날 때마다 인간 연구자에게 다음 단계를 묻지 않습니다. 에이전트끼리 상태를 브리핑하고 새로운 가설을 세워 즉시 다음 실험을 시작하므로, 긴 시간이 소요되는 모델 학습 파이프라인에서 인간의 개입 병목 현상을 제거합니다.
